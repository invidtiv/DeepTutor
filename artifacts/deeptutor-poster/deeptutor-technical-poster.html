Level 1 · adaptive tool surface

Tools are computed per turn

DeepTutor does not hand the model a static toolbox. The visible surface is assembled from learner choices, context flags, active capability tools, and access grants.

User-toggled exploration: brainstorm, web search, paper search, deep reasoning.
Context-mounted skills: retrieval, sources, memory, notebooks, workspace actions, follow-up questions.
Capability-owned tools add a deterministic spine only when that mode is active.
Partner and multi-user grants subtract risky tools before the model ever sees them.

Core insight

The model chooses how to teach, retrieve, reason, consult, and explain. The harness decides what is allowed, what must be recorded, and whether the learner may advance.

Level 2 · capability ownership

Capabilities take over the turn

A capability is a multi-stage tutoring objective. Most reuse the chat loop; specialist pipelines still emit the same result envelope and event stream.

Chat: exploring and responding through one agent loop.
Deep Solve: committed plan, step completion, bounded replans.
Mastery Path: hard per-type gates plus spaced review.

01

Tutoring is loop-native

A turn is a single growing conversation. Tool narration, observations, final answer, and learner clarification all stay inside the same loop.

02

Advancement is computed

Mastery Path asks the engine for the next objective. Pending answers, due reviews, and unmet gates outrank simple stage progression.

03

Retrieval is provider-bound

Each knowledge base keeps its retrieval engine. The service routes to local vector, page, graph, LightRAG, server, or pointer-backed sources.

04

Observability is first-class

The stream protocol records stages, tool calls, tool results, sources, costs, errors, and completion for live UI and replay.

Learning algorithms

Hard mastery gate

Memory and procedure objectives require recency-weighted evidence to clear a 0.90 bar, with early-attempt confidence caps. Concept and design objectives use qualitative Feynman-style assessment.

Type-specific review intervals turn mastery into scheduled reinforcement.
